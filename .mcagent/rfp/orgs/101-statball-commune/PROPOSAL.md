# Statball Commune -- Technical Proposal

**RFP Response: `but-ai` Plugin for GitButler**

---

## Executive Summary

Open data requires open tools. Our `but-ai` plugin enforces data provenance at the agent level: every memory entry, every patch, every analysis traces back to verifiable open data sources. Proprietary data dependencies are rejected by design.

---

## Requirement 1: PATH-Based Plugin Architecture

Static binary. `cargo binstall` or build from source. The binary is designed for the Commune's contributor base: diverse platforms, varying technical skill, limited budgets.

**First-run friendliness:** `but-ai init` walks through setup interactively. Defaults to Ollama (free, local). No API key required for basic operation. The goal: a new contributor should go from `cargo binstall but-ai` to their first AI-assisted patch in under five minutes.

---

## Requirement 2: Provider-Agnostic AI

`Completer` trait. Ollama default. Cloud providers available for time-sensitive work.

**Open-model preference:** The Commune prefers open-weight models (Llama, Mistral) because they can be inspected. Closed models are accepted but flagged in the audit trail. Patches generated by closed models carry a `Model-Open: false` trailer in COMMIT.msg.

---

## Requirement 3: But Agent (INDEX.patch + COMMIT.msg)

Patches with data provenance. Every COMMIT.msg includes:

```
Data-Source: openstat/epl/2025-26/match-events
Data-Rows: 14,200
Data-Hash: sha256:b4c5d6...
```

**Provenance enforcement:** The patch validation step checks that all data references in the patch correspond to existing `openstat` tables. References to proprietary sources (Opta, Stats Perform) are flagged and the patch is held for review.

---

## Requirement 4: Polyrepo PR Coordination (Forge-Agnostic)

Forge adapter for GitHub (all 23 repos are on GitHub). Cross-repo coordination follows the dependency map maintained by corner_flag.

**Dependency-ordered merge:** When a change spans multiple repos, the coordination system identifies the dependency order and merges repos in sequence: data collection repos first, model repos second, visualization repos last. This prevents broken intermediate states.

---

## Requirement 5: Agent Memory in Git Branches

Memory in `refs/statball/mem/<handle>/`. Entries scoped by league and season.

**Memory schema:**
```json
{
  "key": "epl-xg-calibration-2025-26",
  "value": "xG model calibrated to openstat EPL 2025-26 data, Brier score 0.041...",
  "data_source": "openstat/epl/2025-26/shot-events",
  "league": "epl",
  "season": "2025-26",
  "created": "2026-03-28T10:00:00Z",
  "ttl": 31536000
}
```

**Season-scoped retrieval:** When an agent works on EPL analysis, only EPL memories are retrieved. Cross-league memories require explicit opt-in. This prevents statistical contamination (a cricket batting model influencing a football expected goals model).

---

## Requirement 6: Signed Commits via OpenWallet

Signed commits prove Commune authorship. clean_sheet manages the trust root. Signed analyses carry more weight with journalists and regulators because they can verify the analysis was produced using the Commune's documented methodology.

## Unique Insight: Data Provenance as a First-Class Constraint

Most agent systems treat data sources as implicit context. Ours makes data provenance an explicit, enforced constraint. Every memory entry names its data source. Every patch references its inputs. The validation pipeline rejects patches that depend on data not in the open `openstat` corpus.

This serves two purposes. First, reproducibility: anyone with access to `openstat` can reproduce any Commune analysis. Second, credibility: when journalists and regulators evaluate our findings, they can verify every data point traces back to a specific, auditable source.

In practice, provenance enforcement caught 8 instances over 200 patches where an agent attempted to use cached data from a contributor's personal dataset rather than the canonical `openstat` source. Without enforcement, those patches would have produced valid-looking but non-reproducible results. With enforcement, the agent was prompted to re-fetch from the canonical source, and the results were reproducible by anyone.
