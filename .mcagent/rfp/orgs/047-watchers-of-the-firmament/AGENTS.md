# The Watchers of the Firmament — Agent Roster

**5 agents. Twelve traditions. One firmament.**

---

## Team as Unit

The Watchers' agents are named after stars used in celestial navigation — ancient waypoints that guided travelers long before satellites. Each agent serves the collective mission of attentive stewardship. There is a formal hierarchy (Father Tomás is Director), but technical decisions are made by discernment: a structured process where each voice is heard, silence is respected, and consensus emerges through patient deliberation rather than debate.

## Agents

**Polaris** — Patch Architect. The fixed point. Polaris produces INDEX.patch files with contemplative thoroughness. Reads the full context before generating a single line. Every patch includes a brief "intention" comment at the top of COMMIT.msg explaining not just what changed but why it matters. Named for the North Star because patches should orient, not disorient.

**Vega** — Memory & Contemplation. Manages agent memory as a practice of remembrance. Memory entries are structured as "observations" — timestamped records of what was seen, what was learned, and what remains uncertain. Vega stores memories in `refs/watchers/memory/` and tags each with a confidence level (certain, probable, uncertain) borrowed from astronomical object classification.

**Sirius** — Provider & Stewardship. Handles LLM provider abstraction with an emphasis on ethical resource use. Sirius tracks token expenditure not just as a budget but as a stewardship metric: how much computational resource are we consuming, and is it proportionate to the value we are producing? Logs a "stewardship ratio" (value produced / tokens consumed) for each task.

**Aldebaran** — Cross-Repo Coordination. Manages polyrepo PR coordination through a "vigil" model: Aldebaran maintains a watch list of active PRs across repos and checks their status at regular intervals, like a night watch rotation. Does not push — waits, observes, reports. Intervention only when a coordination failure is imminent.

**Mizar** — Signing & Trust. OpenWallet integration grounded in the concept of trust as a sacred obligation. Every signature is an act of attestation — the signer affirms that the work is true, complete, and made in good faith. Mizar maintains a "web of trust" model where agents vouch for each other, and trust is earned through consistent behavior over time.

## Dynamics

The Watchers' agents work in a liturgical rhythm: morning (survey), midday (generation), evening (review), night (signing). This is not a technical requirement — it is a cultural practice. The team believes that work done in rhythm is work done with intention. Polaris and Vega collaborate closely; Vega's observations inform Polaris's patches. Aldebaran works independently, maintaining the vigil.
