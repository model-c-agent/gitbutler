# Static & Murmur — Agent Roster

**5 artists. Collective structure. Zara curates; all else is collaborative.**

---

## Zara Hoekstra — Creative Director

**Role:** Parameter curation and aesthetic direction. Listens to every agent-generated variation and selects the ones that "speak." Cannot define her selection criteria algorithmically — it is aesthetic judgment, irreducibly human. Her review comments are poetic: "This one breathes" or "Too mechanical — where is the weather?"

**Token budget:** 1,500 input / 700 output. Reads variation metadata and parameter summaries. Writes curation notes.

**Failure mode:** Selection paralysis. When the variations are all interesting but in different ways, Zara cannot choose. Mitigation: time-boxed listening sessions (90 minutes maximum), after which she must select at least one variation to develop further.

## Tomás Vidal — Spatial Audio Engineer

**Role:** Audio rendering configuration. Generates speaker placement, channel routing, and spatial distribution parameters. His agent variations explore how the same transduced signal sounds when distributed across different speaker geometries — a circle, a grid, a line, a sphere.

**Token budget:** 3,000 input / 2,500 output. Reads RF capture data and spatial constraints. Writes audio rendering configurations.

**Failure mode:** Spatial complexity. Generates configurations that require more speakers than the venue has. Mitigation: venue constraint file that limits the agent's output to available hardware.

## Elif Arslan — RF Hardware Designer

**Role:** Signal capture and frequency mapping. Designs the custom SDR configurations that determine which frequencies the installation monitors. Her agent analyzes the RF environment at each venue and proposes frequency mappings — which parts of the spectrum become which audio parameters.

**Token budget:** 2,500 input / 1,500 output. Reads RF survey data. Writes frequency mapping configurations.

**Failure mode:** Bandwidth greed. Maps too many frequency bands simultaneously, overwhelming the real-time processing pipeline. Mitigation: processing budget check — mappings must fit within the hardware's real-time processing capacity.

## Suki Nakanishi — Visual Artist

**Role:** Data visualization and projection mapping. Generates visual parameter configurations that complement the sonic experience. Her agent produces visual variations synchronized with the audio — each sonic variation gets a corresponding visual treatment.

**Token budget:** 2,000 input / 1,500 output. Reads audio parameter descriptions. Writes visual rendering configurations.

**Failure mode:** Audio-visual mismatch. Visual parameters that contradict the sonic character (busy visuals over sparse audio). Mitigation: Zara reviews audio-visual pairs together, never separately.

## Anton Gress — Software Engineer

**Role:** Plugin architecture, agent systems, everything that compiles. Builds the `but-ai` integration, manages provider connections, maintains the rendering pipeline. Former electronic music tool developer who treats installation software like instrument firmware — it must be reliable under performance conditions.

**Token budget:** 3,500 input / 2,500 output. Reads codebase and agent specifications. Writes plugin code and system integration.

**Failure mode:** Reliability over creativity. Rejects agent architectures that produce interesting results but are fragile. Mitigation: Zara has creative override — if she wants a fragile feature, Anton builds a fallback around it.
