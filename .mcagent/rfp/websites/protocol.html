Protocol Spec v0.1

WEAVE: How AI Agents Remember

A protocol for AI agents that learn, age, connect, and coordinate through version control.

"What if your AI assistant remembered what it learned yesterday — and knew when to forget?"

The Problem

Three things every AI agent gets wrong

Today's AI agents are powerful but forgetful, isolated, and uncritical of stale knowledge. WEAVE addresses the structural gaps.

Memory Loss

AI agents forget everything between sessions. Insights discovered on Monday are gone by Tuesday. Every conversation starts from zero.

No Coordination

Multi-agent systems can't talk to each other. Two agents working on the same codebase don't know the other exists.

No Aging

Old knowledge never expires. A bug fix from six months ago sits alongside today's architecture, polluting every context window.

The Solution in 60 Seconds

Seven layers, one protocol

WEAVE is a layered protocol. Each layer builds on the one below it. You can adopt as little or as much as you need.

Layer 1 is the foundation: agents produce patches, never modify the working tree directly. Each layer above adds capabilities.

Memory Layer

Memories that live and die

Memories don't expire on a timer. They follow statistical survival curves fitted to actual usage patterns. Knowledge that stops being accessed fades gracefully.

Four distribution families

Exponential for constant-hazard items like bug fixes. Weibull for architecture knowledge that strengthens with age. Bathtub for conventions (risky when new, stable, then eventually outdated). Log-Normal for heavy-tailed cross-repo knowledge.

Fitted, not guessed

Distributions are fitted from actual access history using maximum likelihood estimation, with model selection via AIC. A "surprise index" detects when real usage departs from the model and triggers automatic refitting.

Retrieval

The scoring formula

When an agent retrieves memories, six signals are blended into a single relevance score. Every memory competes for attention on these axes.

  • Motif Resonance 25% — What patterns does it connect to?
  • Classification Match 20% — Where does this belong?
  • Survival Probability 15% — How healthy is this memory?
  • See-Also Distance 15% — What's nearby in the knowledge graph?
  • Tension Urgency 10% — Does this contradict something?
  • Circulation Frequency 10% — How often is this accessed?

Weights are configurable. The composite score is clamped to [0, 1] and results are sorted descending. Motifs with fewer than 3 appearances contribute at 0.3x weight (proto-motifs).

Worktree Integration

Agents in worktrees

Agents work in isolated Git worktrees but share memory instantly through Git refs. No sync protocol needed for co-located agents — the shared .git directory is the gossip layer.

Gossip is free for co-located agents

The full CRDT gossip protocol with vector clocks is designed for cross-repo coordination. Co-located agents in worktrees of the same repo get instant ref visibility for free. No sync needed.

Relaxed CLI contract

Inside a worktree sandbox, agents may use git add and git commit freely. The workspace consistency guarantees are enforced at the integration boundary by worktree_integrate.

Hybrid model

Worktrees for code changes (filesystem isolation, can run tests), refs for memory and coordination (shared visibility, atomic operations). Each mechanism does what it does best.

Orchestration

The 6-phase lifecycle

Every agent task flows through six phases. Tools are gated per phase (least privilege). The last two phases — Catalog and Coordinate — always run, guaranteed by reserved token budgets.

Phase-gated tools

File-write is only available during Implement. Memory-store is only available during Catalog. Each phase loads only the tools it needs, enforcing least privilege.

Budget modes

Full (80%+ remaining): all phases run. Abbreviated (50-80%): reduced validation. Minimum (20-50%): rough implementation only. Emergency (<20%): skip straight to Catalog + Coordinate.

Adoption

Five conformance levels

You don't have to adopt everything at once. Start with Level 1 and grow into the full protocol when you're ready.

1
Patch Producer
Produce valid patches and commit messages. The minimum useful output.
2
CLI Agent
Use the but CLI for all writes. Handle exit codes. Manage sync.
3
Coordinated Agent
Send and receive messages via PR comments. Declare dependencies.
4
Remembering Agent
Store and retrieve memories with survival metadata and narrative references.
5
Full WEAVE
Signed commits, authorization scopes, 6-phase lifecycle, and gossip-based sync.
