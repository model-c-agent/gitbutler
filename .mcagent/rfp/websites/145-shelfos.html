Key Concepts

Card-Catalog Memory

The card catalog → Agent memory index

Memories indexed by five simultaneous classification systems: subject, source, temporal, relational, and hierarchical call number. Multiple paths to the same knowledge.

Circulation Desk

Check out / check in → PR coordination

PRs are library transactions. Agents check out work items, process them, and check them back in. Holds and reserves manage contention.

The Stacks

Library shelves → Git refs by call number

Memory storage organized under refs/catalog/ with a Dewey-like hierarchical addressing scheme. JSON files in Git refs: human-readable, diffable, Git-native.

See Also

Cross-reference cards → Knowledge graph

Bidirectional links between memories capture semantic relationships that keyword search misses. Max 5 per memory forces prioritization of strongest connections.

Deaccession

Library withdrawal → Memory expiration

Memories expire when their TTL lapses and circulation drops. Deaccessioned items move to archive, never deleted. Historical research always possible.

Shelf Reading

Stack verification → Workspace consistency check

but ai shelfread walks the workspace and verifies that actual state matches catalog expectations. Discrepancies surface as "misfiled" items.

Interlibrary Loan

Cross-library lending → Cross-repo coordination

Cross-repo dependencies modeled as loan requests with due dates, reasons, and classifications. Richer than "dependency" because loans have context.

Five-Factor Scoring

Relevance ranking → Memory retrieval

Subject match (35%), call number proximity (25%), see-also distance (20%), circulation frequency (10%), freshness (10%). Every retrieval is scored and ranked.
